### 🍎 设计原则

#### 1. 开闭原则

```
我们的程序要对扩展开放，对修改关闭；我们的程序要给具体使用的时候扩展的接口，但是在具体使用的时候不能让其修改我们的源码， 也就是说我们不用修改源码就能扩展功能，像vue，react等都有扩展的接口。
```

#### 2. 单一职责原则

```
模块只做一件事情，模块的职责越单一越好
```
